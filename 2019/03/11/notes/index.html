    基础知识

    • C++ 所增加的是面向对象的编程（OOP）以及泛型编程。
    • 如果要让窗口一直打开，直到您按任何键，可在 return 语句 cin.get()
    • 通常 main() 被启动代码调用，而启动代码是由编译器添加到程序中的。
    • ANSI/ISO C++ 标准对那些抱怨必须在 main()函数最后包含一条返回语句过于繁琐的人做出了让步。如果编译器到达 main() 函数末尾没有 return ，则默认 return 0。
    • using namespace 这是名称空间编译指令，是 C++ 特性，旨在让您编写大型程序以及将多个厂商现有的代码组合起来的程序时更容易，它还有助于组织程序。按照这种方式，类、函数和变量便是 c++ 编译器的标准组件，它们现在都被放置在名称空间 std 中。仅当头文件没有扩展性 h 时，情况才是如此。
    • c++ 有运算符重载的情况。 << 可以是插入引导作用也可以是左移运算符。 &符号即表示地址运算符，又表示按照AND运算符。 *　既表示乘法，又表示对指针的解引用。
    • 声明通常指出了要存储的数据类型和程序对存储在这里的数据使用的名称。C++没有限制所有的声明变量都位于函数和过程的开始位置。
    • c++ 的自由风格规则将标记间的换行符和空格符看作是可以替换的。
    • C++ 当前之所以如此有吸引力，很大程度是由于存在大量支持 UNIX、Macintosh、Windows编程的类库
    • 编译器倾向于给函数名添加下划线前缀——提示它们对程序具有最后的发言权。
    • C++ 不允许将函数定义在嵌套在另一个函数定义中。每个函数定义都是独立的，所有函数的创建都是平等的。
    • main函数的返回值是返回给操作系统，退出值为 0 则意味着程序运行成功，为非 0 则意味这存在问题。
    • 以两个下划线或下划线和大写字母打头的名称被保留给实现（编译器及其使用的资源）使用。以一个下划线开头的名称保留给实现，用作全局标识符。
    • c++与c99标准有所区别，后者只保证名称中的前 63 个字符有意义。
    • #define 编译指令是 C语言遗留下来的。C++有一种更好的创建符号常量的方法 const。
    • 如果要想 cout 输出 16 进制或者 8进制。可以使用 cout << hex 以及 cout << oct。
    • 虽然 40000 可能表示为 long 的计算机系统，但是 16进制 0x9c40(40000)将被表示为unsigned int。这是因为16进制常用来表示内存地址，内存地址是没有符号的，因此 unsigned int 比 long更适合。
    • C++ 支持的宽字符类型可以存储更多的值，如国际Unicode字符集使用的值 wchar_t。
    • cout.put()函数可以进行输出
    • 如果在声明常量时没有提供值，则该常量的值将是不确定的，且无法修改。
    • c++11 新增了long long，bool，char16_t，char32_t。
    • cin 使用空格、制表符、换行符来确定字符串结束的位置。
    • cin.get()方法会将换行符留在输入队列之中。
    • c++ 允许在声明结构变量时省略关键字 struct
    • 理解共同体就是其中定义的变量，只有一个能起作用。但是结构体只要是你定义的变量都能够起作用。
    • 正常定义一个指针是从栈中分配内存，然而 new() 是从堆中分配内存的。
    • 只能用 delete 来释放使用 new() 分配的内存。然而，对空指针使用 delete 是安全的。
    • 使用静态联编时，必须在编写程序时定义好数组的长度。如若使用动态联编时，程序将在运行时确定数组的长度。
    • 有些编译器将字符串字面值视为只读常量，如果试图修改它们，将导致运行阶段错误。
    • 创建动态结构后，不能将成员运算符句点用于结构名，因为这种结构没有名称，只是知道他的地址。可以用箭头成员运算符(->)。

    • 使用公有派生，基类的公有成员将成为派生类的公有成员；基类的私有部分也将成为派生类的一部分，但只能通过基类的公有和保护方法访问。
    • 有关派生类构造对象的要点如下：1.首先创建基类对象 2.派生类构造函数通过成员初始化列表将基类信息传递给积累构造函数 3.派生类构造函数应初始化派生类新增的数据成员
    • 基类指针可以在不进行显示类型转换的情况下指向派生类对象；基类引用可以在不进行显示类型转换的情况下引用派生类对象
    • 实现多态公有继承，在派生类中重新定义积累的方法或者使用虚方法。
    • virtual 关键字是定义虚方法用的。
    • 如果没有使用 virtual ，程序将根据引用类型或指针类型选择方法；如果使用了 virtual，程序将根据引用或指针指向的对象的类型来选择方法。
    • c++对于virtual虚函数使用的是动态联编。
    • 如果要在派生类中重新定义基类的方法，则将它设置成虚方法；否则，设置为非虚方法。
    • 编译器处理虚函数的方法是：给每个对象添加一个隐藏成员。隐藏成员中保存了一个指向函数地址数组的指针。这种数组称为虚函数表（vbtl）。虚函数表中存储了为类对象进行声明的虚函数的地址。如果派生类提供了虚函数的新定义，该虚函数表将保存新函数的地址；如果派生类没有定义虚函数，该vtbl将保存函数原始版本的地址。如果派生类定义了新的虚函数，则该函数的地址也将被添加到vtbl中。
    • protected ，对于外部世界来说与private 是一样的。对于派生类来说，与 public 一样。
    • 抽象基类（ABC）是当类声明中包含纯虚函数时，则不能创建该类的对象。包含纯虚函数的类只能用作基类。要成为真正的ABC，必须至少包含一个纯虚函数。

      函数原型:int fseek(FILE fp,long offset,int origin);
      函数功能:把fp的文件读写位置指针移到指定的位置.
      ftell函数是用来获取文件的当前读写位置;
      函数原型: long ftell(FILE       fp)
      函数功能:得到流式文件的当前读写位置,其返回值是当前读写位置偏离文件头部的字节数.
    • 关键在于可以将两个函数进行配合，可以用fseek函数把位置指针移到文件尾,再用ftell函数获得这时位置指针距文件头的字节数,这个字节数就是文件的长度.

    Linux基础学习篇

    主机规划与磁盘分区

    • 用来打游戏的游戏计算机所需要的配置一定比办公室用的计算机配置更高档，因为现在一般的三维游戏所需要的 3D 光影运算太多了，所以显卡与CPU资源都会被消耗的非常多。
    • 在 linux 之中，每个设备都被当成一个文件来对待。几乎所有的硬件设备文件都在 /dev 这个目录。
    • IDE 硬盘 /dev/hd[a-d]
    • SCSI/SATA/USB 硬盘 /dev/sd[a-p]
    • U盘 /dev/sd[a-p]
    • 磁盘的第一个扇区主要记录两个重要信息：主引导分区（MBR）446 bytes；分区表 64 bytes
    • 主引导分区是可以安装引导加载程序的地方。MBR 非常重要，因为当系统再开机的时候会主动去读取这个区块的内容，这样系统才会指导你的程序放在哪里该如何进行开机。如果是多系统，MBR这个区块就异常重要。
    • 由于分区表只有 64bytes ，所以硬盘的默认分区表仅能写入四组分区信息。这四组分区信息我们称为主（Primary）或扩展（Extended）分区。
    • 但是既然第一个扇区所在的分区表只能记录四条数据，那我就可以利用额外的扇区来记录更多的分区信息。扩展分区的目的是使用额外的扇区来记录分区信息，扩展分区本身并不能被拿来格式化。
    • 扩展分区最多只能有一个，逻辑分区是由扩展分区持续切割出来的分区。
    • 只有主分区和逻辑分区能够被格式化，扩展分区无法被格式化。如果扩展分区被破坏，所有逻辑分区将会被删除。
    • CMOS 是记录各项硬件参数且嵌入在主板上面的存储器，BIOS 则是一个写入到主板上的一个韧体（再次说明，韧体就是写入到硬件上的一个软件程序）。BIOS是一个在开机的时候计算机系统会主动执行的第一个程序。
    • 这个开机的流程差不多就是 BIOS -> MBR -> boot loader(引导加载程序) -> 内核文件
    • Boot loader 是安装在 MBR 上的程序。它可以提供菜单、载入内核文件、转交其他loader。
    • 引导加载程序除了可以安装在 MBR 之外，还可以安装在每个分区的引导扇区 。loader 只会认识自己的系统分区内的可开机内核文件，以及其他 loader 而已。
    • 所谓的挂载(mount) 就是利用一个目录当成切入点，将磁盘分区的数据放置在该目录下；也就是说，进入该目录就可以读取该分区的意思。
    • 根目录一定要挂载到某个分区，至于其他目录则可依用户自己的需求来挂载。

    Linux 的文件权限与目录配置

    • “用户与用户组”是一个相当健全而且好用的一个安全防护。Linux是一个多用户、多任务的系统，因此可能尝尝会有很多人同时使用这台主机来进行工作的情况发生，为了考虑每个人的隐私权以及每个人喜好的工作环境，因此，这个“文件所有者”就很重要。
    • 权限3个为一组，第一组文件所有者权限，第二组同用户组的权限，第三组其他非本用户组的权限
    • 当然也有相关命令,chgrp 改变文件所属用户组，chown 改变文件所有者，chmod 改变文件的权限。
    • chgrp 要被改变的组名要在/etc/group/内存在才行。他的 -R 参数是表示连同子目录下的所有文件、目录都更新为这个用户组。
    • chown 要被改变的所有者必须已经存在/etc/passwd/之中。 -R同样于上面。
    • cp 命令是会复制执行者的属性和权限的。
    • chmod 是改变权限。-R参数同样于上可以用数字也可以用字母。a 代表全部身份、u是用户、g是用户组、o是其他人。+ 加入、- 减去、= 设置。
    • windows 是否可执行是通过扩展名，例如 .exe这种。而linux是通过权限设置 x 位来进行判断的。
    • 能不能进入某个目录和 r 没有关系，而和 x 有关系。要开放目录给任何人浏览时，应该至少给予 r 和 x。
    • FHS (Filesystem Hiersrchy Standard)目录标准配置，/proc以及/sys本身是不占空间的，因为他们放在了内核当中。
    • /usr 不是 user 的缩写，而是 UNIX software resource的缩写，所以是 unix 操作系统软件资源所放置的目录。
    • /var 是放置常态性变动文件，包括缓存、登陆文件、以及一些软件运行是所产生的文件。
    • uname -r 可以查看内核版本；lsb_release -a 可以查看 distribution 使用的linux 的标准。

    LInux 文件与目录管理

    • . 代表此层目录；.. 代表上层目录;~ 代表“目前用户身份”所在的主文件夹；~account 代表 account 这个用户的主文件夹；- 代表前一个工作目录。
    • mkdir 如果加上 -p 参数，可以字形创建多层目录。-m 可以对目录强加权限设置。
    • rmdir 只能删除空目录，使用 rm -r 可以强制删除目录。
    • 添加系统环境变量的语法：PATH=”$PATH”:/root
    • cp -l -s 都会创建连接文件，但是 -l 是硬连接，-s 是软连接。-s 就相当于一个快捷方式。
    • rm -i 参数互动模式就是会询问用户是否操作；-r递归删除。非常危险啦。-f 就是强制
    • od 以二进制的方式读取文件内容
    • 文件三个主要的变动时间。modification：当该文件的“内容数据“更改时，就会更新这个时间。”status time：”文件权限与属性“被更改是回修改、access time：被使用的时候就会修改。
    • ls 默认显示的文件时间为 mtime。
    • 你新建一个文件，默认权限与 umask 有关。umask 指的是目前用户新建文件或目录时候的权限默认值。
    • SUID 标志所有者，SGID标志用户组。
    • SUID权限仅对二进制文件有效来执行者对于该程序需要具有 X 的可执行权限；本权限尽在执行该程序的过程中有效；执行这将具有该程序所有者的权限。
    • SGID对二进制程序有用；程序执行者对于该程序来说，需具备 x的权限；执行者在执行时将会获得该程序用户组的支持。且SGID可以针对目录。
    • SBIT 当用户对此目录具有w,x权限，即具有写入的权限时；当用户在该目录下创建文件时，仅有自己与root才有权删除该文件。
    • 4为 SUID、2位SGID、1为SBIT。

    Linux磁盘与文件系统管理

    • 第一个扇区最重要，里面有硬盘主引导记录（MBR，446bytes）及分区表（64bytes）。
    • windows 使用的是 NTFS。Linux使用的是 Ext2。默认情况windows是不认识Linux的EXT2.
    • Linux 会将权限与属性放置到 inode 中，至于实际数据则放置到 data block块中。另外还有一个超级块记录整个文件系统的整体信息，包括 inode 与 block 的总量、使用量、剩余量，以及文件系统的格式与相关信息。
    • LInux 属于索引式文件系统。而U盘常用的 FAT格式，是没有 inode 的。所以没有办法将所有block全部读出来，而是每个 block 都记录在前一个 block 当中。因此FAT需要经常进行碎片整理。
    • 若文件小于block的大小，则该block剩下的空间啊就不能在使用了。
    • inode 本身不记录文件名，而是放在block中的。

    • export 命令就是干这个用的。

    X86汇编语言从实模式到保护模式

    • 处理器的引脚有很多，其中一部分使用来将参与运算的数字送入处理器内部。有些引脚是复用的。
    • 指令F4H表示让处理器停机，当处理器取得并执行这条指令后就停止工作。
    • 并非每一个二进制数都代表着一条指令。每种处理器在设计的时候，也只能拥有有限的指令，从几十条到几百条不等。一个处理器就能够识别的指令的集合，称为该处理器的指令集。
    • 8086中，CS是代码段寄存器，DS是数据段寄存器，ES是附加段寄存器。当需要在程序中同时使用两个数据段时，DS指向一个，ES指向另一个。如果没有指定，那么默认是DS。SS是栈段寄存器。
    • IP是指令指针寄存器，它只和CS一起使用，而且只有处理器才能直接改变他的内容。CS和IP共同形成逻辑地址。
    • 8086处理器的逻辑分段，起始地址都是16的倍数，被称为是按16字节对齐的。
