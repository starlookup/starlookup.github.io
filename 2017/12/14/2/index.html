《程序员的自我修养》读书笔记一

readingbin
  • bin
  • rading

　　这本书还是因幡师傅推荐给我的，看到swing师傅blog里也写过这本书读书笔记，想想自己也来写一个吧


个人理解有误之处，望师傅们指正

温故而知新

  • 书籍基于兼容x86指令集的32位CPU个人计算机

一、硬件设备

关键部件

1.CPU

2.内存

3.I/O控制芯片

相关知识

  • 早期计算机CPU频率与内存一致，连同一根总线，I/O速度较内存与CPU更慢，为了能让CPU与I/O通信，每个设备都有相应的控制器
  • 后来CPU频率不断提高，内存跟不上CPU，产生了与内存频率一致的系统总线，CPU以倍频方式与系统总线通线。这就是产生了CPU倍频。
  • 再后来图形化界面普及，图形芯片要与CPU和内存大量交换数据，且对速度有一定要求。便产生了北桥芯片（处理高速）与南桥芯片（处理低速）。
  • 再后来到了近代，CPU频率没有大幅提升了，限制在了4GHz，人们就想增加CPU数量来提高整体速度。最常见的SMP(对称多处理器)，即每个CPU地位功能完全相同。
  • 但是多处理器成本比较高，于是人们就共享昂贵的缓存部分，保留多个核心，以一个处理器外包装。产生了多核处理器。

注：速度提高与CPU数量不能成正比，因为一个程序不能分成没有任何关系的若干个问题。

二、系统软件

分类

1.平台性（系统内核、驱动程序、运行库等）
2.用于程序开发（编译器、汇编器、链接器等）

相关知识

  • 系统软件体系从上到下可分为 software->runtime library->operating system kernel->hardware
  • 每层之间要通信，协议叫接口。两层之间由下层定义借口，上层使用。
  • 虚拟机技术是在硬件与操作系统之间增加了一层虚拟层，使得可以同时运行多个操作系统。
  • 应用程序与运行库之间通信的接口为API，不同运行库API不同，如32位windows提供API被称为Win32
  • 为了尽量利用CPU资源，现在采用了多任务系统，应用程序以进程方式运行，权限比操作系统低。进程地址空间独立。
  • CPU由操作系统分配，根据优先级高低，所谓抢占式，一个进程运行时间不可过长，不然会被暂停，分配给其他进程。
  • 多任务导致内存可能不够，于是增加中间层，就如虚拟机机制一样，提出了虚拟地址，可以通过映射转成物理地址。
  • 分隔思想有分段与分页，后面会详细写。

三、线程

概念

程序执行流最小单元，由线程ID、当前指令指针、寄存器集合、堆栈组成。可以理解一个进程由一到多个线程组成。线程共享相关的内存空间。

相关知识

  • 多线程优势：

    1. 有效利用等待时间，比如等待网络响应时可以执行其他操作。
    2. 复杂进程，可分工进行
    3. 有些程序本身要求多线程，比如BT下载
    4. 发挥多核的计算能力
    5. 相较于多进程，多线程在数据共享方面效率更高

  • 线程与进程共享的数据：程序代码（.text）、全局变量、局部静态变量（.data）和堆上数据（malloc）、打开的文件，A线程打开的文件可由B线程读写；

  • 线程私有数据：局部变量和函数参数、TLS(线程局部存储)
  • 线程数量大于处理器数量时，则就一定会有单处理器对应多线程，处理器让每个进程仅执行很短的时间（毫秒），造成表面的同时执行。

  • 线程分三种状态：

    • 1.执行
    • 2.就绪：线程可以立即运行，但CPU被占用
    • 3.等待：线程无法执行

  • 等待结束->就绪->就绪结束->执行->执行结束->就绪或等待

  • 线程调度方式，大部分都参照优先级调度与轮转法。轮转发就是每个线程都执行一会。优先级调度就是每个线程有自己的优先级，优先级高的全部执行完后，优先级低的才能执行。
  • 在Windows和Linux优先级可手动设置，系统也会根据线程表现自动调整，比如频繁等待的线程优先级就较容易提高。
  • 饿死指的就是，一个线程总是有优先级比它高的线程在执行，以至于它始终无法执行。系统为了解决这个问题会提高它的优先级。
  • linux不如windows有明确的线程与进程，linux中所有的执行（进程、线程）都叫任务，每个任务类似与一个单线程的进程。不过多个任务共享内存空间时，就类似于多线程的进程。

线程安全

  • 多线程同时访问一个共享数据，可能造成线程竞争
    • 举个例子全局变量i = 1;

  • 线程1: ++i;

  • 线程2: –i;

  • 每个线程都是（1）取数据到寄存器（2）完成计算（3）存储回i,两个线程同时运行结果可能为0,1,2。

  • 为了防止线程同时访问，所以提出了锁的概念，即一个线程访问时获取锁，访问结束后释放锁。使其他线程在它访问时不能访问。

  • 有了锁也不一定安全，例如由于优化和并发，产生过度优化。

  • 举个例子全局变量x = 0
  • 线程一：lock();x++;unlock();
  • 线程二：lock();x++;unlock();
    *正常来看结果应为2，但是由于编译器为了提高x的访问速度，把x放到了寄存器中，而不同线程寄存器是独立的，线程一操作完后，因为后面还要访问x,没有将运算后的结果返回给x，导致线程二读取的时候x还是0
  • 用户使用的是存在于用户态的多线程，而不是内核线程。用户态的线程与内核线程有三种模型：
    • （1）一对一模型:缺点数量受内核进程控制，效率低。
    • （2）多对一模型：缺点一个用户线程阻塞可能导致其他都无法执行,但是效率提高
    • （3）多对多模型
